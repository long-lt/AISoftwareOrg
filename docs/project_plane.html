Lộ Trình Triển Khai Chi Tiết (Nhấp để mở rộng)

Nhấp vào từng giai đoạn trong timeline để xem chi tiết các nhiệm vụ kỹ thuật, thời hạn và sản phẩm bàn giao tương ứng.

Phase 1: Tinh chỉnh Cơ chế Điều phối Agent (LangGraph Core)
Q1 - Hoàn thành

Xây dựng lõi đa tác nhân hoạt động bất đồng bộ bằng LangGraph state machine, tích hợp phân cấp RBAC và cost tracking.

Nhiệm vụ cốt lõi:
  • Thiết lập BaseAgent class và cơ chế cấp quyền rành mạch giữa các agents (READ, WRITE, EXECUTE, CRITICAL).
  • Hoàn thành bộ theo dõi chi phí CostTracker cộng dồn token LLM thực tế quy đổi USD.
  • Viết suite kiểm thử tự động gồm 26 unit tests bao phủ 100% các tính năng lõi.
Phase 2: Xây dựng Giao diện Dashboard Quản trị Tích hợp
Q2 - Đang triển khai

Phát triển bảng quản trị SPA full-stack (FastAPI Backend + Vite & Vanilla JS Frontend) với khả năng hiển thị thời gian thực.

Nhiệm vụ cốt lõi:
  • Hoàn thiện hệ thống REST APIs trên FastAPI hỗ trợ xem trạng thái pipeline, cost tracking, log stream và model switcher.
  • Tạo giao diện dashboard tối ưu hóa trải nghiệm người dùng với kiến trúc glassmorphism, chủ đề tối cao cấp (premium dark theme).
  • Tích hợp SSE (Server-Sent Events) để truyền log hoạt động của agent theo dạng stream trực tiếp lên terminal dashboard.
Phase 3: Sandbox Hóa & Agent Thử Nghiệm Runtime (Mobile Sandbox)
Q3 - Sắp diễn ra

Cô lập quá trình biên dịch ứng dụng Flutter, kích hoạt emulator ảo tự động để phát hiện crash dump và phân tích giao diện UI.

Nhiệm vụ cốt lõi:
  • Thiết lập môi trường biên dịch an toàn chạy bên trong Docker container cô lập đối với mã nguồn Dart sinh ra.
  • Phát triển Runtime Verification Agent điều khiển headless Android/Web emulator để chạy thử ứng dụng di động.
  • Sử dụng OCR và phân tích ảnh chụp màn hình để phát hiện lỗi tràn bố cục giao diện (layout overflow) tự động.
Phase 4: Tự học Nâng cao & Tự động Đóng gói ZIP Bàn giao
Q4 - Lập kế hoạch

Mở rộng cơ sở dữ liệu kinh nghiệm tự rút ra sau các lần sửa code lỗi, tích hợp quy trình phê duyệt tri thức HITL và nén mã nguồn.

Nhiệm vụ cốt lõi:
  • Xây dựng phân hệ kinh nghiệm lưu vết (Self-Learning Experience DB) để ghi nhận những tệp lỗi biên dịch và các dòng code vá thành công.
  • Hoàn thiện dashboard duyệt tri thức (HITL Approval Queue) giúp quản trị viên lọc, phê duyệt các kinh nghiệm chạy task của agent.
  • Đóng gói quy trình tự động zip toàn bộ mã nguồn sạch của Flutter MVP + API Backend để bàn giao ngay khi pipeline báo PASS.
