Quy Trình Chế Tạo Ứng Dụng 12 Bước (12-Phase Flutter Pipeline)

Phân hệ flutter_ai_factory vận hành luồng khép kín đi từ specs nghiệp vụ đến khi đóng gói mã nguồn sạch đã compile thành công:

01

BA Spec Analysis

Phân tích yêu cầu thô từ ngôn ngữ tự nhiên để sinh Specs nghiệp vụ chi tiết.

02

Architect Blueprint

Thiết kế cấu trúc thư mục Clean Architecture chuẩn của dự án di động.

03

API Spec Definition

Thiết kế chi tiết cấu trúc dữ liệu Model và các endpoints REST APIs kết nối.

04

Backend Coding

Viết các API Controller, mock data backend tích hợp trên Node.js hoặc Python.

05

UIUX Component Design

Lên kế hoạch và bố cục các thành phần giao diện, nút bấm, bảng màu.

06

Developer Dart Coding

Sinh mã nguồn Flutter chi tiết cho tất cả các Views, Providers và Model Dart.

07

API Integration

Kết nối màn hình Flutter với các API Backend đã xây dựng ở các bước trước.

08

QA Static Analysis

Thực thi lệnh phân tích tĩnh flutter analyze để thu thập danh sách lỗi.

09

Refactor/Repair Loop

Vòng lặp vá lỗi tự động viết đè sửa code cho đến khi static check báo PASS.

10

Runtime Validation

Mở ứng dụng ảo, chạy test unit runtime và lưu giữ dump log, crash report.

11

Security Compliance

Quét các package dependencies xem có chứa lỗ hổng bảo mật nghiêm trọng nào không.

12

Reviewer Export

Tổng hợp toàn bộ quá trình, nén ZIP mã nguồn sạch bàn giao cho khách hàng.

Vai trò Agent READ (Đọc file) WRITE (Ghi file) EXECUTE (Chạy lệnh) CRITICAL (Đóng gói/Git)

Planner / Architect Y N N N
Dev Agent / Refactor Agent Y Y N N
QA Agent / Runtime Agent Y N Y N
