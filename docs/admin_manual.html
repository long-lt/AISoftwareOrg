Operator & Admin Manual

Hướng dẫn vận hành chi tiết dành cho Quản trị viên của Hệ thống AI Software Factory

1. Giới thiệu tổng quan 2. Quản lý HITL Queue 3. Phê duyệt Checkpoints 4. Đăng ký & Cấu hình LLM 5. Quản lý Workspace & ZIP

1. Giới thiệu tổng quan vận hành

Chào mừng bạn đến với cẩm nang vận hành dành cho **Platform Owner** và **Engineering Manager**. Với vai trò Quản trị viên, bạn nắm giữ quyền kiểm soát tối cao đối với toàn bộ luồng sinh ứng dụng tự động, bảo mật phân quyền tác nhân, cấu hình các nhà cung cấp mô hình ngôn ngữ lớn (LLM Providers) và kiểm duyệt tri thức tự học của hệ thống.

Dashboard quản trị giao tiếp qua REST APIs thời gian thực kết nối tới lõi điều phối LangGraph để cung cấp khả năng quan sát (observability) sâu sắc, theo dõi ngân sách token và can thiệp thủ công khi cần thiết.

Thông tin cơ bản: FastAPI backend chạy mặc định tại cổng 8000. Dashboard quản trị SPA Vite phục vụ trực tiếp tại địa chỉ http://localhost:8000 trong chế độ Production hoặc http://localhost:5173 trong chế độ phát triển (Developer mode).

2. Quản lý Hàng đợi Kinh nghiệm tự học (HITL Experience DB)

Trong quá trình vá lỗi mã nguồn Flutter (Repair Loop), Refactor Agent sẽ rút ra các kinh nghiệm thực tế về việc dòng code Dart nào bị lỗi static analyze và cách viết đè sửa đổi thành công. Các tri thức này được gửi vào hàng đợi duyệt trước khi nạp chính thức vào cơ sở tri thức chung (Experience Knowledge Base).

Các bước phê duyệt kinh nghiệm:

1
Truy cập hàng đợi: Điều hướng đến mục "Knowledge Hub" hoặc "HITL Queue" trên thanh menu trái của Dashboard để hiển thị danh sách các tri thức đang chờ phê duyệt.
2
Đánh giá nội dung: Nhấp xem chi tiết từng dòng kinh nghiệm. Kiểm tra phần error_pattern (lỗi static check cũ) và fix_pattern (mã nguồn sửa đổi của Dev Agent).
3
Quyết định phê duyệt: Nhấn nút "Approve" để hệ thống nạp kinh nghiệm này vào tệp cơ sở dữ liệu SQLite chung giúp tối ưu hóa độ chính xác của agent cho các phiên sinh code tiếp theo, hoặc nhấn nút "Reject" để hủy bỏ nếu code sửa đổi không tối ưu hoặc vi phạm tiêu chuẩn coding.
Cảnh báo bảo mật: Chỉ phê duyệt các kinh nghiệm có phần code sửa đổi sạch, tường minh. Tránh phê duyệt các sửa đổi chứa mã độc hoặc import các thư viện không an toàn từ bên ngoài.

3. Phê duyệt Checkpoints tác vụ nhạy cảm

Đối với các tác vụ mang tính nhạy cảm như ghi đè tệp tin cốt lõi, chạy các tập lệnh thực thi shell trên server hoặc chuẩn bị merge mã nguồn vào nhánh chính (Main branch), hệ thống LangGraph sẽ tự động kích hoạt chốt chặn **Checkpoint Interceptor**, tạm dừng tiến trình và gửi thông báo chờ phê duyệt.

Cách thức xử lý Checkpoints:

  • Khi một dự án chuyển sang trạng thái "Waiting for Approval" (màu vàng nhấp nháy trên Dashboard), nhấp vào dự án đó để xem tiến trình chi tiết.
  • Kiểm tra phần ghi nhật ký hành động (Action Logs) để biết chính xác Agent nào đang yêu cầu thực hiện hành động gì và tác động tiềm ẩn đến dự án.
  • Sử dụng API POST /api/checkpoints/{id}/approve (hoặc nhấn nút tương ứng trên UI) để phê duyệt cho Agent tiếp tục chạy, hoặc từ chối để hủy tiến trình và quay về checkpoint an toàn trước đó.

4. Đăng ký & Cấu hình mô hình ngôn ngữ lớn (LLM Setup)

Hệ thống hỗ trợ cơ chế thay đổi động cấu hình LLM trực tiếp tại thời điểm chạy mà không cần khởi động lại Server API.

Các thao tác quản lý LLM:

  • Xem danh sách nhà cung cấp: Vào mục "AI Models" để hiển thị danh sách các LLM Providers đang khả dụng (như OpenRouter, OpenAI, Gemini).
  • Thay đổi LLM Provider chính: Gửi yêu cầu POST /api/providers/{name}/use để kích hoạt nhà cung cấp mong muốn. Dashboard sẽ tự động đồng bộ và nạp cấu hình mới.
  • Kiểm soát ngân sách token: Theo dõi kỹ biểu đồ phân bổ chi phí tokens tại trang chủ Dashboard. Nếu chi phí vượt định mức cho phép của dự án, hãy cấu hình chuyển đổi sang sử dụng các mô hình gọn nhẹ hơn như gpt-4o-mini hoặc gemini-1.5-flash.

5. Quản lý Workspace & Tải mã nguồn ZIP

Khi một pipeline chế tạo ứng dụng di động báo trạng thái hoàn thành thành công (PASS toàn bộ 12 phases), hệ thống sẽ cô lập mã nguồn sạch cuối cùng nằm tại thư mục workspace/ của dự án tương ứng.

Các bước xuất bản mã nguồn:

1
Xác minh biên dịch sạch: Kiểm tra xem vòng lặp vá lỗi (Repair Loop) đã báo PASS hoàn toàn đối với lỗi Dart tĩnh chưa.
2
Tải mã nguồn ZIP: Nhấp vào dự án trên Dashboard và chọn nút "Export source code". Thao tác này kích hoạt API GET /api/jobs/{slug}/download để nén toàn bộ mã nguồn sạch của ứng dụng di động Flutter và companion REST API backend thành tệp tin ZIP tải về ngay lập tức.
