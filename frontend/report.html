Tái Cấu Trúc Đơn Vị Giao Diện & Cơ Chế Kết Nối API

Đơn vị phát triển: AI Software Factory Team · Trạng thái: Đóng gói thành công

1. Tổng quan Kiến trúc Tái cấu trúc

Nhằm đáp ứng yêu cầu làm sạch mã nguồn, loại bỏ trạng thái nguyên khối (monolithic) và chuyển dịch sang hướng mô-đun phát triển mở rộng lâu dài, chúng tôi đã tái thiết kế thành công toàn bộ hệ thống giao diện 19 màn hình của AI Software Factory Platform.

Các cải tiến cốt lõi:

  • index.html (Rút gọn 90% dòng code): Từ file gốc 1000 dòng code phức tạp, hiện tại chỉ còn 130 dòng đóng vai trò làm khung chứa cố định (Sidebar, Top Header, Footer) và cung cấp một cổng hiển thị động (Dynamic Viewport) dạng <main id="main-view-viewport"></main>.
  • Tách biệt Template hiển thị (src/views/): Toàn bộ cấu trúc HTML của 19 màn hình đã được tách biệt thành các file .html độc lập trong thư mục src/views/. Các tệp này được nạp động thời điểm build qua tính năng ?raw của Vite.
  • Tách biệt Logic điều khiển (src/js/views/): Mỗi màn hình được điều phối bởi một Lớp (Class) xử lý kế thừa từ lớp cơ sở BaseView, tự trị quản lý sự kiện, đồng bộ API và giải phóng bộ nhớ.

2. Cơ chế Tích hợp Kết nối API Realtime

Kiến trúc mới cung cấp một cơ chế tích hợp API cực kỳ nhất quán, mượt mà và bảo mật từ Frontend đến hệ thống API Backend thông qua 3 lớp xử lý:

A. Lớp Cấu hình Ủy quyền Development Proxy (Vite server)

Khi chạy phát triển cục bộ, Vite Server được cấu hình một cổng Proxy trong file vite.config.js để tự động chuyển tiếp toàn bộ các yêu cầu mạng từ Frontend lên Server API Backend (FastAPI chạy ở cổng 8000) nhằm tránh các lỗi về CORS:

// Cấu hình Proxy trong vite.config.js
proxy: {
  '/api': {
    target: 'http://127.0.0.1:8000',
    changeOrigin: true,
    secure: false,
  }
}

B. Lớp API Client Tập trung (src/js/api.js)

Toàn bộ mã nguồn kết nối mạng được tập trung hóa trong api.js. Tại đây, client tự động đính kèm Token xác thực JWT lấy từ LocalStorage (được lưu tại tab Pipelines hoặc Settings) vào tiêu đề Authorization: Bearer <token> cho mỗi request:

// Cơ chế tự động đính kèm JWT Token trong src/js/api.js
export async function fetchJSON(url) {
  const headers = {};
  const token = getToken(); // Lấy từ localStorage
  if (token) headers['Authorization'] = 'Bearer ' + token;
  const resp = await fetch(url, { headers });
  if (!resp.ok) {
    throw new Error(`Truy cập ${url} thất bại: ${resp.status}`);
  }
  return resp.json();
}

C. Vòng đời Tự động Đồng bộ trên View (Lifecycle-based Polling)

Mỗi View Class (ví dụ OverviewView, PipelinesView) tận dụng các hàm vòng đời để quản lý việc gọi API:

  1. onMount(): Được gọi ngay khi View được chèn vào DOM. Thực hiện gọi API lấy dữ liệu lần đầu và thiết lập bộ hẹn giờ setInterval(() => this.onUpdate(), 5000) để tự động reload dữ liệu mỗi 5 giây.
  2. onUpdate(): Thực thi tải dữ liệu bất đồng bộ từ các hàm trong api.js và ánh xạ (render) trực tiếp vào các phần tử giao diện bằng việc cập nhật văn bản, vẽ biểu đồ tròn, biểu đồ SVG, hoặc sinh các dòng logs.
  3. onUnmount(): Được gọi khi người dùng chuyển sang màn hình khác. Giải phóng hoàn toàn bộ đếm thời gian (clearInterval), giúp trình duyệt tối ưu hóa bộ nhớ, tránh tình trạng gọi API ngầm lãng phí băng thông và CPU.

3. Sơ đồ Luồng Dữ liệu API động

1. Người dùng bấm chọn Menu Sidebar / Chuyển hướng trang
↓
2. Router kích hoạt showView() → Gọi view.onUnmount() của View cũ
↓
3. Gỡ bỏ HTML cũ → Chèn Template view.html mới → Gọi view.onMount()
↓
4. view.onMount() gọi api.js lấy dữ liệu ban đầu + Thiết lập setInterval
↓
5. Dữ liệu API phản hồi → view.onUpdate() cập nhật giao diện (realtime 5s)
↓
6. Rời khỏi trang → view.onUnmount() dọn dẹp bộ đếm interval sạch sẽ

4. Minh họa Code Kết nối trong View thực tế

Dưới đây là cấu trúc code tiêu biểu trong src/js/views/PipelinesView.js thể hiện sự phối hợp hoàn hảo giữa giao diện và cơ chế đồng bộ API:

import BaseView from './BaseView.js';
import template from '../../views/pipelines.html?raw';
import * as api from '../api.js';

export default class PipelinesView extends BaseView {
  constructor(router) {
    super('pipelines', template);
    this.router = router;
    this.refreshInterval = null;
  }

  onMount() {
    // 1. Gọi dữ liệu API lần đầu
    this.onUpdate();

    // 2. Thiết lập cơ chế tự động đồng bộ hoá realtime mỗi 5 giây
    this.refreshInterval = setInterval(() => this.onUpdate(), 5000);
  }

  onUnmount() {
    // 3. Huỷ bỏ timer để giải phóng bộ nhớ khi rời trang
    if (this.refreshInterval) {
      clearInterval(this.refreshInterval);
      this.refreshInterval = null;
    }
  }

  async onUpdate() {
    try {
      // Gọi API bất đồng bộ lấy danh sách tác vụ từ backend
      const data = await api.getTasks();
      
      // Ánh xạ dữ liệu trả về trực tiếp vào phần tử giao diện
      const target = this.container.querySelector('#task-summary');
      if (target) {
        target.innerHTML = `<div class="total">Total Tasks: ${data.total}</div>`;
      }
    } catch (err) {
      console.error("Lỗi cập nhật API:", err);
    }
  }
}

5. Tích hợp Dynamic Models & Ô Tìm kiếm nhanh cho AI Agents

Để tăng cường tính linh hoạt và cá nhân hoá cho từng Agent, chúng tôi đã tích hợp thành công cơ chế cấu hình LLM Provider động cùng ô tìm kiếm nhanh (autocomplete datalist) gốc:

A. Luồng cấu hình liên màn hình (Cross-View Config Flow)

Thông tin khóa API và địa chỉ Base URL được người dùng cấu hình một lần duy nhất tại màn hình AI Models (lưu trữ an toàn trong localStorage dưới dạng api_key_<provider> và base_url_<provider>).

Khi người dùng chỉnh sửa cấu hình một Agent tại màn hình AI Agents:

  • Ứng dụng tự động đọc Provider được cấu hình cho Agent đó (ví dụ: openrouter, openai, deepseek, v.v.).
  • Gọi API động /api/models?provider=...&key=...&base_url=... để nạp danh sách models thực tế thời gian thực tương ứng với thông tin tài khoản của Provider đó.

B. Giao diện Dropdown tìm kiếm nhanh (Searchable Autocomplete Dropdown)

Thay thế thẻ <select> tĩnh cũ bằng cấu trúc HTML5 native cực kỳ nhẹ và mượt mà, hỗ trợ gõ lọc tìm kiếm ngay lập tức:

<!-- Ô nhập tìm kiếm liên kết với Datalist trong src/views/agents.html -->
<input type="text" id="modal-field-model" list="modal-field-model-list" placeholder="Gõ để lọc..." required>
<datalist id="modal-field-model-list"></datalist>

Khi chọn thay đổi LLM Provider, hệ thống lập tức dọn dẹp ô nhập cũ, tự động gửi request tải lại danh sách model mới và cập nhật tức thì danh sách gợi ý trong <datalist> đi kèm phản hồi trạng thái trực quan.

6. Kết luận

Kiến trúc **Modular Vanilla JS** mới kết hợp cùng cơ chế **Vite ?raw Template** và **Lớp API Client tập trung** đã mang lại một bước tiến vượt bậc cho dự án:

  1. Giúp mã nguồn tách biệt, sạch sẽ, tuân thủ đúng triết lý Separation of Concerns.
  2. Tránh rò rỉ bộ nhớ hoàn toàn bằng việc quản lý vòng đời timer tối ưu.
  3. Dễ dàng gắn thêm bất kỳ Endpoint API Backend mới nào chỉ bằng việc khai báo hàm trong src/js/api.js và gọi cập nhật tại phương thức onUpdate() của View tương ứng.
  4. Cơ chế liên kết và đồng bộ cấu hình qua localStorage giúp ứng dụng hoạt động thông minh, mượt mà và tối đa hóa trải nghiệm người dùng mà không cần thiết lập database phức tạp ở Frontend.
